Тестове завдання: Data Enricher & Dispatcher

Реалізуйте невеликий сервіс на Go, який:
1. Отримує дані з API A
2. Обробляє ці дані згідно з умовами
3. Відправляє певні елементи в API B
4. Має механізм повторної спроби (retry) в разі невдачі при POST-запиті

Деталі реалізації

1. Отримання даних:
   - GET-запит до https://jsonplaceholder.typicode.com/users

2. Обробка:
   - Якщо email користувача закінчується на .biz — надсилаємо в API B
   - Інакше — виводимо повідомлення в консоль

3. Відправка в API B:
   - POST-запит (можна використати https://webhook.site або мок)
   - Тіло запиту:
     {
       "name": "<ім’я користувача>",
       "email": "<email>"
     }
   - Якщо POST-запит не вдався (не 2xx) — зробити retry до 3 разів із затримкою

4. Структура коду:
   - client/api_client.go
   - model/user.go
   - service/dispatcher.go
   - main.go

Бажано (але не обов’язково):

- Використання context.Context
- Конфігурація URL через .env або flags
- Логування результатів
- Юніт-тести (наприклад, на фільтрацію .biz)

Оцінювання:

- Читабельність і структура коду
- Коректність логіки
- Retry-механізм
- Обробка помилок
- Тести, логування (як бонус)
